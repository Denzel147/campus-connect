# 7-Minute Run-of-Show + Speaking Roles

Goal: Hit all rubric points with crisp visuals. Practice transitions.

## Roles
- Gedeon (UX/UI): Overview, Design choices, Feedback integration (2:00)
- Frontend Dev: Implemented features, Code quality/testing (2:00)
- Backend Dev: Database & Architecture (2:00)
- PM/QA: Challenges, Next steps, References, close (1:00)

## Minute-by-Minute
- 0:00–0:20 (Gedeon) — Title & Overview
  - Problem, solution, who we are
- 0:20–1:20 (Frontend Dev) — Implemented Features & Tools
  - Planned vs implemented; justify stack
- 1:20–2:20 (Backend Dev) — Database & Data Management
  - ERD, storage/retrieval, security
- 2:20–3:30 (Backend Dev) — System Architecture
  - Diagram, patterns, scalability
- 3:30–4:15 (Frontend Dev) — Code Quality & Testing
  - Standards, examples, test approach
- 4:15–5:00 (PM/QA) — Technical Challenges & Solutions
  - Top 2 challenges; outcomes
- 5:00–5:30 (Gedeon) — Feedback Integration
  - What changed after feedback
- 5:30–6:30 (PM/QA) — Next Technical Steps
  - Milestones; owners
- 6:30–7:00 (PM/QA) — References & Close

## Slide Cues
- Keep each slide to 1 key message; speaker notes include metrics/examples
- Use high-contrast theme; readable fonts; light/dark mode consistent

## Q&A (3 minutes)
- Likely questions
  - Why not use framework X? Trade-offs
  - How do you secure user data? Hashing, RBAC, HTTPS
  - Scalability approach? Pagination, caching, CDN
  - Accessibility? Color contrast, keyboard nav, alt text
